상시 운영용 배포 안내

이 프로젝트는 이제 Render 같은 외부 호스팅에 올리면,
내 PC를 켜두지 않아도 24시간 접속 가능한 웹사이트로 운영할 수 있습니다.

자동 반영 방식
- GitHub 저장소의 연결된 브랜치에 코드가 올라가면
- Render가 자동으로 다시 빌드하고 배포합니다.
- 따라서 로컬 터미널이나 로컬 서버를 켜둘 필요가 없습니다.

현재 프로젝트 상태
- Render용 render.yaml 포함
- Node start 스크립트 포함
- 상태 확인용 /api/health 포함
- 로컬 임시 공유용 cloudflared 배치파일은 별도 유지

Render에서 만들 서비스 유형
- Web Service

권장 연결 방식
1. 이 폴더를 GitHub 저장소에 업로드
2. Render에서 해당 저장소 연결
3. render.yaml 기반으로 배포
4. 발급된 onrender.com 주소 사용

배포 후 기대 결과
- 누가 언제 열어도 접속 가능
- 내가 코드 수정 후 GitHub에 반영하면 자동 재배포
- 로컬 PC를 끄더라도 사이트 유지

중요 참고
- Render 무료 플랜은 슬립 상태가 있을 수 있음
- 완전한 상시 응답이 중요하면 유료 플랜 권장
